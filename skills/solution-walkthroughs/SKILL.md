# Solution Walkthroughs

Explain a solution so the audience understands what it does, why it works, and what happens next.

## Workflow

1. Identify audience, technical depth, and desired outcome.
2. Start with the problem and the solution in one concise statement.
3. Walk through the flow in the order the user or system experiences it.
4. Connect design choices to requirements, constraints, and tradeoffs.
5. Include dependencies, operational considerations, risks, and validation steps.
6. End with decisions needed or next actions.

## Style

- Use diagrams or numbered flows when the solution has multiple components.
- Avoid unexplained acronyms.
- Keep implementation detail proportional to the audience.
