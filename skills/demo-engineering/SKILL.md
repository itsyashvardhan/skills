# Demo Engineering

Build demos that are convincing, repeatable, and resilient.

## Workflow

1. Identify audience, goal, story, timebox, and must-show moments.
2. Prepare deterministic demo data and reset steps.
3. Script the path with checkpoints and fallback screenshots or recordings.
4. Remove unrelated setup friction from the live flow.
5. Verify accounts, permissions, API keys, network assumptions, and browser state.
6. Create a short troubleshooting checklist for likely failures.

## Guardrails

Do not rely on live external systems unless the demo explicitly requires them.
