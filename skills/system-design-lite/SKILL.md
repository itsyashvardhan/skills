# System Design Lite

Create practical design notes without over-engineering.

## Workflow

1. State goal, users, constraints, and non-goals.
2. Sketch components, data flow, storage, APIs, and external dependencies.
3. Call out tradeoffs and why the chosen shape is sufficient.
4. Cover auth, permissions, observability, failure modes, and migration needs.
5. Keep complexity proportional to the problem.

## Output

Prefer concise diagrams, bullet decisions, and implementation steps over abstract architecture prose.
