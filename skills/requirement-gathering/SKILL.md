# Requirement Gathering

Convert unclear goals into actionable requirements.

## Workflow

1. Identify stakeholders, users, business goal, constraints, and success criteria.
2. Separate current state, desired state, assumptions, dependencies, and open questions.
3. Ask only questions that materially reduce delivery risk.
4. Define functional requirements, non-functional requirements, edge cases, and out-of-scope items.
5. Convert requirements into testable acceptance criteria.
6. Track unresolved decisions explicitly.

## Output Patterns

- Use user stories when product behavior is user-facing.
- Use process steps when the work is operational.
- Use acceptance criteria with concrete examples when implementation will follow.
